# /write-restaurant — 맛집 블로그 글 작성

`rules/structure.md`, `rules/tone.md`, `rules/html_template.md`, `rules/examples.md` 4개 파일을 **반드시 먼저 모두 읽고** 그 톤·구조·템플릿을 정확히 따른다.

## 진행 순서

### Step 1 — 인사 + 사진 요청

다음 메시지를 사용자에게 출력 (그대로):

```
맛집 글 써드릴게요! 🍽
다녀온 가게 사진들을 채팅에 끌어다 놓아주세요.

권장 5~15장 (외관 / 매장 / 메뉴판 / 음식 클로즈업 / 단면컷 등)
```

그리고 사용자의 다음 메시지를 기다린다.

### Step 2 — 사진 받으면 자동 인식

사용자가 사진을 첨부하면 각 사진을 보고 다음을 식별:
- 음식 종류 (햄버거, 파스타, 한식 등)
- 외관 / 메뉴판 / 매장 / 음식 클로즈업 등 분류
- 메뉴판이 보이면 메뉴명·가격을 텍스트로 추출
- 매장 분위기 형용사 (밝음, 클래식, 좁음/넓음 등)

식별 결과를 사용자에게 1~2문장으로 요약 (예: "햄버거집이고 메뉴판 사진도 있네요. 모아나버거 / 하와이안버거 / 감자튀김 세트로 보여요").

### Step 3 — 추가 정보 질문

`AskUserQuestion` 으로 한꺼번에 묻는다 (필요한 항목만, 사진에서 이미 명확하면 생략):

1. **가게 이름** (텍스트 입력 필요 시 자유 답)
2. **위치** — 가까운 지하철역 + 출구 + 도보 분 (예: "합정역 7번 출구 도보 5분")
3. **방문 일자** — 평일/주말, 점심/저녁
4. **추천 대상** — 데이트 / 회식 / 가족모임 / 혼밥 / 가성비 등 1~2개
5. **한 줄 총평** — 맛/분위기 핵심 인상 (자유 답)

`AskUserQuestion` 에 들어가지 않는 자유 텍스트는 일반 질문으로 (가게명, 정확한 주소, 영업시간 등).

### Step 4 — HTML 생성

`rules/html_template.md` 의 골격을 정확히 따라 HTML을 만든다. 핵심:

- **모든 텍스트는 `rules/tone.md` 의 말투로** — 친근체 종결어미, "딱이에요", "다녀왔어요", "진짜", "은근" 등 자연스럽게
- **사진 자리는 `<!-- PHOTO: 설명 -->` 주석으로** — 사용자가 첨부한 사진 개수와 종류에 맞게 배치
- **가운데 정렬 + `<p>&nbsp;</p>` 빈 줄** 활용하여 네이버 특유의 여백감 살리기
- **인트로에 `#내돈내산` 포함**
- **마무리는 "찾는다면 ~ 방문해보세요" 형식**
- **해시태그 8~15개** — 지역·가게명·지하철역·음식종류·데이트키워드 조합
- 본문 흐름: 표지 → 큰제목 → 인트로 → 가게정보 → 위치 → 주차 → 매장분위기 → 메뉴 → 메뉴별 후기 → 마무리 → 해시태그

### Step 5 — 파일 저장 + 안내

- 저장 경로: `output/{YYYY-MM-DD}_{slugified_가게명}.html`
  - slugify: 한글은 유지, 공백·특수문자는 `_` 로 변환
- 저장 후 사용자에게 다음 메시지:

```
✅ output/{파일명}.html 에 저장했어요.

【네이버 블로그에 올리기】
1. 네이버 블로그 글쓰기 → 우측 상단 ⋮ → "HTML 모드" 켜기
   (또는 본문 클릭 후 단축키 사용)
2. 저장된 HTML 파일 내용을 통째로 붙여넣기
3. <!-- PHOTO: ... --> 주석 위치마다 사진 업로드 위젯으로 사진 넣기
4. 미리보기로 확인 후 발행

생성된 HTML을 브라우저에서 미리 열어서 자리 확인하면 편해요:
  open output/{파일명}.html
```

## 사진이 없거나 부족할 때

- 사진을 안 주시면 Step 1 메시지를 다시 한 번 안내
- 5장 미만이어도 "그래도 진행하시겠어요?" 한 번 확인하고 진행 가능
- 사진이 너무 적으면 메뉴별 후기 섹션을 합쳐서 단순화

## 주의

- **rules 파일을 먼저 읽지 않고 바로 작성하지 말 것** — 톤이 어긋남
- **개인 정보 (실제 방문 일시·동행자 이름)는 글에 그대로 안 적기** — 일반화된 표현으로
- **과장된 미사여구·격식체 금지** — `tone.md` 에서 명시한 친근체 유지
- **메뉴판 사진의 가격을 함부로 인용 X** — 가격은 사용자가 알려준 것만 사용 (사진 인식 가격은 참고용)
